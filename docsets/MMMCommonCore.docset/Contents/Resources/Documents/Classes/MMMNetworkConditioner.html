  • -conditionBlock:inContext:estimatedResponseLength:

    If simulation of errors and delays is turned on for this app, then the block will be called on the main queue after a random delay and possibly with an error object (with its error message/code/domain being the same in all cases). In case the simulation is turned off, then the block is called immediately with error being nil.

    Usage example from MMMPublicLoadableImage: \code [[MMMNetworkConditioner shared] conditionBlock:^(NSError *error) { [self dispatch:^{ if (error) { [self didFailWithError:error]; } else { self->_image = image; self.loadableState = MMMLoadableStateDidSyncSuccessfully; } }]; } inContext:NSStringFromClass(self.class) estimatedResponseLength:data.length ]; \endcode
